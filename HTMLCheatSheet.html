Lists:

Ordered Lists:

Lists where elements are sequentially ordered.

  • Opened with < ol > closed with < /ol >
  • Each element opened with < li > closed with < /li >

Example:

Race Results

  1. Billy
  2. Alice
  3. David

Unordered Lists:

Lists where elements are unordered.

  • Opened with < ul > closed with < /ul >
  • Each element opened with < li > closed with < /li >

Example:

Shopping List

  • Eggs
  • Apples
  • Avacados

Definition Lists:

Lists where elements are being defined.

  • Opened with < dl > closed with < /dl >
  • Each element to be defined is opened with < dt > closed with < /dt >
  • Each definition of an element is opened with < dd > closed with < /dd >

Example:

Links:

Links are created using < a href="weblink" > TEXT < /a >

Other Sites:

HTML:

< a href="http://www.imdb.com" > IMDB < /a >

Actual:

IMDB

Same Site Pages:

Use the realitive path of the page

HTML:

< a href="OtherTestPage.html" > Other Test Page < /a >

Actual:

Other Test Page

Link Notes:

  • "mailto:" - add this to the front of the href string if its an email
    • HTML: < a href="mailto:boi@boi.boi" > boi@boi.boi < /a >
    • Actual: boi@boi.boi

  • "target" - add this as an attribute with the value "_blank"
    • HTML: < a href="http://www.imdb.com" target="_blank" > IMDB < /a >
    • Actual: IMDB

  • Linking to a specific part of a page: Must give a header an attribute: id="idname"
    • HTML: < a href="#top" > Top < /a >
    • Actual: Top

  • To link to a specific part of a different page, add #linkName to the end of the link
    • Ex: Hit the second link on this page...

Tables:

Table Tags:

  • < table > - contains the table data
  • < tr > - contains the data of a single row of the table
  • < td > - contains the data of a single cell of a row of the table
  • < th > - contains the heading for a single column of the table

Table Tag Attributes:

  • colspan (td/th) - how many column cells this cell will encompass
  • rowspan (td) - how many row cells this cell will encompass

Long Table Tags:

  • < thead > - headings of the table
  • < tbody > - body of the table
  • < tfoot > - footer of the table

Example:
Heading 1 Heading 2 Heading 3
Row 1 data1.(1/2) data1.3
Row 2 data(2/3).1 data2.2 data2.3
Row 3 data3.2 data3.3
Foot Row (4) data4.1 data4.2 data4.3

Forms:

< form > - overarching tag for forms

Attributes:

  • action - a URL for the page to recieve the server info when it is submitted
  • method - (get/post) get - value added to end of URL | post - values sent in HTTP Headers (for larger data)
  • id - for unique identification by scripts

< input > - used to create form controls

  • type - the kind of input
    • "text" - creates a single line of text input (name, maxlength)
    • "password" - creates same thing but blacks out text(name, size/maxlength)
    • "radio" - select only one option (name, value, checked)
    • "checkbox" - select multiple options (name, value, checked)
  • name - the name of the info being entered
  • maxlength - the maximum length of characters that may be entered
  • value - indicates the value that will be sent to the server if checked
  • checked - indicates which (if any) bo should already be "checked"

< select > - makes a drop down list (use with < form >)

Attributes and Tags:

  • name - name of the form
  • < option > - specify options user can select from
    • value - the name of what will be sent to the server
    • selected - what begins selected
    • multiple - allows user to select multiple things from the list
    • size - how many options to show at once (mostly for multiple)

< input > - for uploading files

Attributes:

  • type - indicates the input type
    • "file" - creates a text box with a "browse" button
    • "submit" - sends the form to the server
    • "image" - uses an image as your submit button (must use src="filepath" with it)
    • "hidden" - won't show to user, allows web page to submit extra data user doesn't control
  • name - name the attribute (if you want to)
  • value - controls the text that appears on the button

HTML5: Form Validation

Use required="required" under the < input > tag to make the user submit something there

HTML5: Date Input

Use type="date" under < input > to ask the user for a date

HTML5: Email and URL Input

Use type="email" for emails, type="url" for urls

HTML5: Search Input

Use type="search" under < input > to make it a search box

Attribute: name="wordsBelow" content="related info"

  • "description" - description of page
  • "keywords" - keywords of page seperated by commas
  • "robots" - "noindex" if search engines shouldn't pick it up, "nofollow" if they shouldnt link to pages it links to
